## 1. The goal ‚Äî pure machine search (high level)

The north star: the default runtime path is **selected by BubbleBeam and generated by tinygrad/codegen**, not a
hand-written whole kernel. A route is "pure" only when the winning primitive is expressible in the search space,
BubbleBeam can generate/select it, the evaluator proves correctness + lifecycle performance, and the hand-owned
kernel is no longer required for default performance. The aim is not more search over existing knobs ‚Äî it is to
**expose the missing low-level capability**, then let the machine schedule competitive kernels.

There are three levels of "pure," and naming them matters:
1. **Hand-ASM** (`extra/qk_owned_flash_decode.hip`) ‚Äî the 100% baseline (llama.cpp parity).
2. **Hand-authored UOp tile ‚Üí tinygrad codegen** ‚Äî the ISA is machine-generated, the UOp *structure* is hand-written.
   This is where the decode-attention candidate is today.
3. **BubbleBeam generates the kernel structure** ‚Äî the true end state.

Model/hardware: Qwen3-8B-Q4_K_M, AMD RX 7900 XTX (gfx1100, RDNA3, wave32).

## 3. The exact issue (drilled down, measured)

The 12.8√ó whole-decode gap @ctx4096 **is the attention tile**, not the combine:
- Per-kernel @ctx4096: `flash_block_tiled` **3711¬µs** vs `flash_state_combine` ~20¬µs + `flash_state_gmax` ~11¬µs
  (**~124√ó**; the combine is ~0.8% of the cost ‚Äî a red herring).
- The tile is ~3.7ms √ó 36 layers ‚âà the whole 149ms/token step; owned's attention is ~290¬µs/layer.

Owned-vs-generated parity (`bench/qk-owned-oracle-parity/latest.json`): **8 MATCH / 5 MISMATCH / 1 UNKNOWN**.
- MATCH (primitives present + correct): `v_dot2` (packed dot), cross-lane reduce (`ds_bpermute` 5-step butterfly),
  **LDS 8192 staging**, coalesced loads, route_bound, token-match. **There is NO missing primitive.**
- MISMATCH (all scheduling-driven): `vgpr` 88 vs 64, `waitcnt` 50 vs 21, `latency_shadow_fill` 3.75 vs 0.2,
  `split_kv_combine`, `wd_tok_s`.

**The cycle budget pins the cause.** gen **281** cyc/token vs owned **22** cyc/token. The tile's inner loop does, per
token: a packed dot (2 `v_dot2`), a 5-step `ds_bpermute` cross-lane reduce, then the online-softmax merge
(max/exp/rescale of running `m,l,acc`). 5 `ds_bpermute` @ ~40-cyc LDS latency, **fully exposed**, ‚âà 200 cyc ‚âà the
whole gen per-token cost. **Owned overlaps those 5 reduces across tokens (cross-iteration); the generated tile
exposes them.** The serial online-softmax recurrence (`mx/acc/den.after(tt)` in the kernel) is what blocks the
overlap; tinygrad's `linearize` is a topo-sort with **no instruction scheduler / software pipeliner**, so the
recurrence-bound reduce latency is emitted in-order and fully exposed.

In modulo-scheduling terms: **II = max(RecMII, ResMII)**. comgr already pipelines the K/V global loads (ResMII is
hidden). The bound is **RecMII** ‚Äî the loop-carried online-softmax recurrence fed by the per-token cross-lane reduce.

## 5. My hypothesis on closing the gap

**The gap is exposed cross-iteration `ds_bpermute` reduce latency, and the fix is a codegen Layer-3 modulo
scheduler** that overlaps iteration N+1's *independent* work (load, `v_dot2` dot, the cross-lane reduce ‚Äî none of
which depend on the running `m,l,acc`) into the shadow of iteration N's *serial* online-softmax merge, **while
preserving the unroll** (the unroll's registers buy the ILP that already helps; do not undo it ‚Äî that's what the
no-unroll and split-score refutations proved).

Why codegen, not a kernel rewrite: the kernel-level split-score attempt pipelined the reduces but lost the unroll's
interleaving and regressed. The scheduler must operate *below* the kernel structure ‚Äî reorder the already-unrolled,
already-linearized work so independent reduces issue into the recurrence's latency shadow, at a target initiation
interval.

This builds on existing foundations:
- **Layer 1 ‚Äî latency-aware list scheduler** (built): `extra/qk_codegen_list_scheduler.py`, `SCHED_LIST=1`. Reorders
  the linearized UOp list intra-iteration. (Refs: Gibbons-Muchnick '86; Hennessy-Gross '83.)
- **Layer 2 ‚Äî recurrence unroll** (built): `extra/qk_codegen_recurrence_unroll.py`, `SCHED_UNROLL=8`. Creates the
  cross-iteration copies (the ILP raw material).
- **Layer 3 ‚Äî modulo scheduler** (MISSING, the build): cross-iteration software pipelining. (Refs: Lam '88 modulo
  scheduling + MVE; Rau '94/'96 iterative modulo scheduling; Allan '95 survey.)

Two architectural arms:
- **Arm A (build first) ‚Äî UOp-level modulo pass** in `tinygrad/codegen/late/linearizer.py` (`def linearize` at
  `:7`; the **exact hook model** is the `SCHED_LIST` insertion at `linearizer.py:49-52` ‚Äî
  `if getenv("SCHED_LIST"): newlst = list_schedule(newlst)` right after the topo-sort builds `newlst`). Env-gated +
  cache-keyed, modeled on the opt-in lowering gates in `tinygrad/codegen/__init__.py` (e.g. `SCHED_UNROLL` at `:54`,
  `COALESCED_LOAD_LOWERING` at `:93`, `V_DOT2_LOWERING` at `:133`), and on `extra/qk_fdot2_lowering.py` /
  `extra/qk_warp_reduce_lowering.py`. Partition each loop body into {serial-recurrence via the `.after()` carry,
  independent}; schedule iter N+1's independent ops into iter N's `lgkmcnt` shadow at a target II; MVE-rename the
  staged buffer for depth-2 double-buffering if needed.
- **Arm B (fallback) ‚Äî tinygrad's own ISA scheduler** on the `Ops.INS` path (`extra/qk_asm_scheduler.py`, dormant;
  `tinygrad/renderer/amd/elf.py`). This is the deeper, comgr-independent answer.

**The key risk / open question for the scope: does comgr re-schedule and *undo* the UOp-level ordering?** If the
hotloop diff shows the emitted ISA ignores Arm A's order, that finding **forces Arm B**. The scope must include this
as an explicit decision point, not assume Arm A survives to the ISA.

Honest caveat: this might hit the abstraction limit. Owned proves the hardware does 94 tok/s @ctx4096, so the
distance is *physically* closable ‚Äî but if a properly built modulo scheduler still can't approach comgr/hand-ASM on
the RecMII recurrence, that is the terminal `SEARCH_BLOCKED_BY_CODEGEN__SCHEDULING` finding. Scope it to reach that
verdict honestly if it comes.
